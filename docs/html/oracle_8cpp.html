Description:
In this node is implemented the oracle action of the rosplan. This node allows to knwo if the current complete and consistent hypothesis is the winning one or not. This is done by comparing the value contained in the client of oracle service and the client of the hypo_found service, that contains the ID of the current complete and consistent hypothesis. The client of the service winhypo is only use to know the who, what and where associated to the current ID. If the two IDs matches then the plan is finished, instead if not, a replanning is needed to continue the investigation.
